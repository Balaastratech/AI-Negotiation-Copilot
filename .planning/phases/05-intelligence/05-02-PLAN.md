<task type="auto">
  <name>Create useNegotiation hook</name>
  <files>frontend/hooks/useNegotiation.ts</files>
  <action>
    Create `frontend/hooks/useNegotiation.ts`.
    Connect `WebSocketClient`, `AudioWorkletManager`, `MediaStreamManager`, and `NegotiationContext`.
    Handle ALL ServerMessageType messages via WebSocket client:
      - binary frame (ArrayBuffer/Blob) -> `audioManager.playChunk`
      - `CONNECTION_ESTABLISHED` -> dispatch `SET_CONNECTED`
      - `CONSENT_ACKNOWLEDGED` -> dispatch `SET_CONSENTED`
      - `SESSION_STARTED` -> dispatch `SET_NEGOTIATING`
      - `TRANSCRIPT_UPDATE` -> dispatch `APPEND_TRANSCRIPT`
      - `STRATEGY_UPDATE` -> dispatch `SET_STRATEGY`
      - `AI_RESPONSE` -> dispatch `APPEND_TRANSCRIPT` (with speaker: 'ai')
      - `OUTCOME_SUMMARY` -> dispatch `SET_OUTCOME`, dispatch `SET_NEGOTIATING` to false
      - `AUDIO_INTERRUPTED` -> `audioManager.clearQueue()` MUST be called synchronously to instantly flush all pending audio buffers and silence AI.
      - `SESSION_RECONNECTING` -> dispatch `SET_ERROR` with a non-blocking status message (frontend connection to FastAPI remains open to hide the 10-min backend handoff).
      - `AI_DEGRADED` -> dispatch `SET_DEGRADED`
      - `ERROR` -> dispatch `SET_ERROR`
    Expose API: `connect(wsUrl)`, `grantConsent(version, mode)`, `startNegotiation(context)`, `endNegotiation(finalPrice, initialPrice)`, `sendFrame(base64)`, and current `state`.
    AVOID: Adding UI component logic inside this hook. It should only manage state dispatch and service/SDK orchestration.
  </action>
  <verify>cd frontend; npx tsc --noEmit hooks/useNegotiation.ts</verify>
  <done>Hook compiles without TypeScript compiler errors.</done>
</task>
